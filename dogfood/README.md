# TEREO Dogfood

This is the live proof that the repo uses its own rules.
It stays small on purpose.

## Law

Dogfood keeps only what still holds under noise.

Dogfood is not the archive.
Dogfood is the current loop.

## Loop

1. make a promise about the skill or runtime
2. run a real check
3. keep only what earns a receipt
4. log keep, discard, or weak proof

## What Lives Here Now

- `promises/`
  The current self-promises.

When dogfood runs for real, it should leave:

- a receipt
- a kept baseline or a discarded patch
- a short result trail

## Rule

If dogfood stops increasing trust, it is failing.
